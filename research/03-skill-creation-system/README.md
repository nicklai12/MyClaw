# 對話式技能建立系統設計

## 1. 核心理念

NanoClaw 的技能 (Skills) 是給開發者在 Claude Code CLI 中使用的 SKILL.md 檔案。
LINE 版本的創新在於: **讓終端使用者直接透過 LINE 對話建立技能，完全不需要寫程式碼。**

## 2. 什麼是「技能」？

在這個系統中，技能 = **一組指令 + 觸發條件 + 記憶**

```
技能範例:
┌─────────────────────────────────────────┐
│ 名稱: 每日天氣報告                        │
│ 觸發: 每天早上 8:00                       │
│ 指令: 查詢台北天氣，用簡潔的方式告訴我       │
│       今天要不要帶傘                       │
│ 輸出: Push Message                       │
└─────────────────────────────────────────┘

技能範例:
┌─────────────────────────────────────────┐
│ 名稱: 摘要助手                            │
│ 觸發: 使用者說「幫我摘要」                   │
│ 指令: 將使用者接下來傳送的文字/連結摘要成     │
│       3-5 個重點                          │
│ 輸出: Reply Message                      │
└─────────────────────────────────────────┘
```

## 3. 技能建立流程 (對話式)

### 3.1 自然語言建立

使用者只需要用自然語言描述需求:

```
使用者: 我想要每天早上收到今天的待辦事項提醒
AI: 了解！我幫你建立「每日待辦提醒」技能。
    確認一下:
    ⏰ 觸發時間: 每天早上 8:00
    📋 功能: 提醒你今天的待辦事項

    你的待辦事項要從哪裡來？
    [1] 直接告訴我，我幫你記住
    [2] 從 Google Calendar 讀取
    [3] 其他

使用者: 1

AI: 好的！技能已建立 ✅
    「每日待辦提醒」- 每天 08:00 推送

    你可以隨時跟我說:
    - 「加入待辦：買牛奶」來新增待辦
    - 「今天的待辦」來查看
    - 「修改提醒時間」來調整
```

### 3.2 技術實現

每個技能在後端儲存為一個簡單的 JSON + Markdown:

```json
// users/{userId}/skills/daily-todo.json
{
  "id": "daily-todo",
  "name": "每日待辦提醒",
  "trigger": {
    "type": "cron",
    "value": "0 8 * * *"
  },
  "prompt": "讀取使用者的待辦事項清單 (在 memory.md 的 ## 待辦事項 區塊)，用友善的方式提醒今天要做的事情。",
  "created_at": "2026-02-07T10:00:00Z",
  "status": "active"
}
```

```markdown
<!-- users/{userId}/memory.md -->
# 我的記憶

## 待辦事項
- [ ] 買牛奶
- [ ] 回覆客戶信件
- [ ] 下午 3 點開會
```

### 3.3 觸發類型

| 類型 | 說明 | 範例 |
|------|------|------|
| `cron` | 定時執行 | 每天 8:00, 每週一 |
| `keyword` | 關鍵字觸發 | 使用者說「幫我摘要」 |
| `pattern` | 正則匹配 | 使用者傳送 URL 時 |
| `always` | 每次對話都執行 | 回應風格設定 |
| `manual` | 手動觸發 (Rich Menu) | 按鈕觸發 |

## 4. AI 如何動態產生技能

### 4.1 技能生成 Prompt

當使用者描述需求時，AI 使用以下流程：

```
System Prompt:
你是一個技能建立助手。使用者會用自然語言描述他們想要的功能。
你需要:
1. 理解使用者的需求
2. 確認細節 (觸發方式、頻率、輸出格式)
3. 生成技能配置 (JSON)
4. 生成技能的執行指令 (prompt)
5. 儲存到使用者的技能資料夾

技能配置格式:
{
  "name": "技能名稱",
  "trigger": { "type": "cron|keyword|pattern|always", "value": "..." },
  "prompt": "給 AI 的指令，描述這個技能要做什麼",
  "tools": ["web_search", "memory_read", "memory_write"],
  "output_format": "text|flex_message"
}
```

### 4.2 技能執行流程

```
觸發條件符合
    ↓
載入技能配置 (JSON)
    ↓
組合 prompt:
  - 技能指令 (prompt)
  - 使用者記憶 (memory.md)
  - 當前上下文 (時間、地點等)
    ↓
呼叫 Claude API
    ↓
格式化輸出 → 發送回 LINE
```

## 5. 預設技能範本

為了讓使用者快速開始，提供以下預設範本:

| 範本名稱 | 描述 | 觸發 |
|----------|------|------|
| 智慧摘要 | 傳連結自動摘要 | URL 偵測 |
| 翻譯助手 | 自動翻譯訊息 | 關鍵字「翻譯」 |
| 每日天氣 | 推送天氣預報 | 每天 7:00 |
| 記帳助手 | 記錄花費 | 關鍵字「花了」 |
| 提醒事項 | 定時提醒 | 關鍵字「提醒我」 |
| 新聞摘要 | 每日新聞重點 | 每天 8:00 |

## 6. 技能管理介面

透過 LINE Rich Menu 和對話:

```
使用者: 我的技能
AI: 📋 你目前有 3 個技能:

    1. ✅ 每日天氣報告 (每天 07:00)
    2. ✅ 智慧摘要 (偵測到 URL 時)
    3. ⏸️ 記帳助手 (已暫停)

    你可以:
    [編輯] [暫停/啟用] [刪除] [新增技能]
```

## 7. 進階: 技能互相調用

技能可以引用其他技能的資料:

```
「每週回顧」技能:
- 讀取「記帳助手」的本週花費紀錄
- 讀取「待辦事項」的完成狀況
- 生成週報
```

這透過共享的 `memory.md` 實現，每個技能都可以讀寫特定區塊。

## 8. 技術考量

### 8.1 安全性
- 技能的 prompt 由 AI 生成，需要 sanitize
- 使用者間的記憶完全隔離 (不同 userId)
- 限制技能可用的工具 (例: 不允許執行系統命令)

### 8.2 成本控制
- 每個技能執行都是一次 API call
- 定時技能需要控制頻率 (最小間隔 1 小時)
- 可以設定每月 token 上限

### 8.3 可靠性
- 技能配置儲存在 SQLite + 檔案系統
- 執行失敗自動重試 (最多 3 次)
- 錯誤通知使用者

## 9. 結論

對話式技能建立是這個專案最大的差異化特點。它讓非技術使用者也能:
- 用自然語言描述需求
- AI 自動轉換為可執行的技能
- 透過 LINE 管理所有技能
- 不需要寫任何程式碼

這本質上是把 NanoClaw 的 Claude Code Skills 概念，從「開發者工具」轉化為「終端使用者功能」。
